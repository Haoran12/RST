# 2026-03 开发日志

## 增量更新（2026-03-04）- LLM 请求伪装对齐（OpenAI Compat + Anthropic/Gemini/DeepSeek）

### 后端
1. Provider 基础层新增统一浏览器风格出站请求头与 SillyTavern 标识，并提供统一 headers 合并入口；默认附带 `User-Agent`（含 `SillyTavern` 标识）、`Origin/Referer`、`Sec-Fetch-*`、`Sec-CH-UA*` 等字段：`backend/app/providers/base.py`。
2. 新增出站敏感请求头脱敏函数 `redact_outbound_headers()`，用于日志上下文中隐藏密钥（`Authorization/x-api-key/api-key/x-goog-api-key`）：`backend/app/providers/base.py`。
3. OpenAI 兼容链路（`openai/openai_compat/deepseek`）请求上下文补齐脱敏后的 headers 字段，保持 Chat Completions 载荷与 SillyTavern OpenAI-compatible 形态一致：`backend/app/providers/openai.py`。
4. Anthropic 链路对齐 SillyTavern 关键格式：`POST /messages` + `anthropic-version/x-api-key/Content-Type`，并在 provider 请求上下文中记录脱敏 headers：`backend/app/providers/anthropic.py`。
5. Gemini 链路对齐 SillyTavern 关键格式：`POST /models/{model}:generateContent?key=...` + `Content-Type`，并在 provider 请求上下文中记录脱敏 headers 与脱敏 query：`backend/app/providers/gemini.py`。

## 增量更新（2026-03-05）- Scene 提取与 Status Panel 在场人物刷新修复

### 后端
1. `SceneService.parse_scene_tag()` 由“多标签取第一条”调整为“多标签取最后一条”，避免同一回复中存在多个 `<scene>` 时读取到旧场景：`backend/app/services/scene_service.py`。
2. 场景人物分隔符扩展为 `,` / `，` / `、` / `;` / `；`，提升 `<scene>` 中 `characters` 解析兼容性：`backend/app/services/scene_service.py`。
3. 更新场景解析单测，覆盖“取最后标签 + 全角冒号 + 混合分隔符”行为：`backend/tests/test_scene_service.py`。

### 前端
1. 聊天成功追加 assistant 消息后，`chat store` 主动广播 `rst-lore-data-changed`（`source=chat-store`），驱动 `StatusPanel` 自动重拉 scene/人物数据，不再依赖手动刷新：`frontend/src/stores/chat.ts`。
2. `StatusPanel` 在场人物匹配由“仅角色主名精确匹配”升级为“主名 + aliases + role 的归一化匹配”，并支持弱包含匹配，修复 `<scene>` 人物有别名/修饰词时不更新的问题：`frontend/src/components/StatusPanel.vue`。

### 测试与验证
1. 后端回归通过：`python -m pytest backend/tests/test_scene_service.py backend/tests/test_chat_flow.py -q`（10 passed）。
2. 前端类型检查通过：`npm run type-check`（workdir: `frontend`）。
