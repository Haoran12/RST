## 增量更新（2026-03-07）- RST Lores 全 SQLite 收口与 JSON 快照导入导出

### 后端
1. `LoreStore` 默认改为“旧会话首次访问自动迁移到 SQLite，之后固定使用 SQLite”，不再对运行态 Lore 数据回退 JSON；迁移完成后会清理 `rst_data/default/*.json`、`rst_data/characters/*.json`、`scene_state.json`、`scheduler_template.json`、`.index/index.json` 等残留文件：`backend/app/storage/lore_store.py`。
2. 新增 `SQLiteLoreStore`，以 `lore.db` 作为 RST Lore 运行态唯一存储，并补齐 Lore 条目、角色、形态、记忆、SceneState、SchedulerTemplate、索引与 FTS 检索实现：`backend/app/storage/sqlite_lore_store.py`、`backend/app/storage/migration/schema.sql`。
3. 新增 JSON 快照导入/导出能力，迁移工具改为复用同一条数据链路：`JsonLoreStore.export_json_bundle()` 导出快照，`SQLiteLoreStore.import_json_bundle()` 导入快照，`JsonToSqliteMigrator` 基于快照完成旧 JSON 会话迁移，避免维护并行写入逻辑：`backend/app/storage/json_lore_store.py`、`backend/app/storage/sqlite_lore_store.py`、`backend/app/storage/migration/migrator.py`。
4. 新会话初始化调整为仅创建 `rst_data/lore.db`，不再生成运行态 Lore JSON 文件：`backend/app/services/session_service.py`。
5. `Lores` 路由新增 JSON 快照导入/导出接口：`POST /sessions/{session_name}/lores/import-json` 与 `GET /sessions/{session_name}/lores/export`；导入后会刷新 scene runtime cache，并重置 scheduler 的命中缓存：`backend/app/routers/lores.py`、`backend/app/services/lore_service.py`。

### 前端
1. `RST Lores Panel` 顶部操作区新增会话名旁的 `Import Static Lore` / `Export JSON` 按钮，导出当前会话的 RST Lore 快照 JSON：`frontend/src/components/panels/RstLorePanel.vue`。
2. 导入入口增强为自动识别两类 JSON：
   - `rst-lore-snapshot-v1` 快照 JSON：直接导入到 SQLite 并刷新面板；
   - 旧版静态 Lore JSON：继续走现有转换导入流程。
   相关改动位于：`frontend/src/components/panels/RstLorePanel.vue`、`frontend/src/api/lores.ts`、`frontend/src/types/lore.ts`。
3. 新增中英文文案，覆盖“导出 JSON”“快照导入成功”“快照导出成功”等提示：`frontend/src/i18n/messages.ts`。
